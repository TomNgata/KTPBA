Inferred Tournament Structure

Each match week, teams are paired on adjacent bowling lanes (odd lane vs even lane). Each pairing plays three competitive formats in sequence on the same evening. Each format is best of 3 games, and winning a format earns the team 1 standings point. Maximum 3 points available per team per matchday.

Singles1 player bowls 3 individual gamesEvery team player must bowl ≥1 singles game across the season7:00–8:00 PM1 for match winner
Doubles2 players bowl 3 games, alternating framesEvery team player must bowl ≥1 doubles game across the season8:10–9:10 PM1 for match winner

Baker's Style — Rules Engine

In Baker's Style, a team of 5 players shares a single 10-frame game. Each player bowls exactly 2 frames in a fixed rotation (Player 1: frames 1&6, Player 2: frames 2&7, Player 3: frames 3&8, Player 4: frames 4&9, Player 5: frames 5&10). The 10th frame is bowled by Player 5. Lineup order can be changed between games (not during). Best of 3 games: first team to win 2 games wins the Teams match.

✓
Tie-Break Rule — Confirmed If both teams bowl identical pin totals in a game, a roll-off game is played immediately to determine the winner of that game. The roll-off result is recorded as a separate game entry in the games table with is_rolloff = true. The roll-off winner takes that game's win in the best-of-3 count. This applies to all three formats (Singles, Doubles, and Teams/Baker).
⚠
Privacy Rule (Critical) Team player identities and lineups must NEVER be displayed on the public-facing frontend. Player participation tracking is backend-only (Supabase). Leaderboards show team standings only. This is to prevent competitive intelligence between opponents.

-- Standings view (Supabase SQL)
CREATE OR REPLACE VIEW team_standings AS
SELECT
  t.id, t.name, t.slug,
  COUNT(fm.id) FILTER (WHERE fm.winner_team_id = t.id) AS total_points,
  COUNT(fm.id) FILTER (WHERE fm.format = 'singles' AND fm.winner_team_id = t.id) AS singles_pts,
  COUNT(fm.id) FILTER (WHERE fm.format = 'doubles' AND fm.winner_team_id = t.id) AS doubles_pts,
  COUNT(fm.id) FILTER (WHERE fm.format = 'teams' AND fm.winner_team_id = t.id) AS teams_pts,
  COALESCE(SUM(g.home_score) FILTER (WHERE mu.home_team_id = t.id), 0) +
  COALESCE(SUM(g.away_score) FILTER (WHERE mu.away_team_id = t.id), 0) AS total_pinfall
FROM teams t
LEFT JOIN format_matches fm ON TRUE
LEFT JOIN matchups mu ON fm.matchup_id = mu.id
  AND (mu.home_team_id = t.id OR mu.away_team_id = t.id)
LEFT JOIN games g ON g.format_match_id = fm.id
WHERE t.is_active = TRUE
GROUP BY t.id, t.name, t.slug
ORDER BY total_points DESC, total_pinfall DESC;

Admin Score Entry — Flow Detail
  1. Admin logs in via Supabase Auth → lands on /admin dashboard
  2. Selects active matchup (tonight's games pre-populated)
  3. Selects format: Singles → enters Game 1 home pin total and away pin total
  4. System auto-determines game winner (higher score) and displays it
  5. Admin enters Game 2; system checks if format is decided (2-0) or continues to Game 3
  6. After all 3 formats done: system calculates which team wins each format → adds 1 standing point each
  7. Admin optionally logs player participation (backend-only dropdown)
  8. Admin optionally enters Baker lineup for teams games (backend-only)
  9. Admin clicks "Publish Result" → scores go live on public pages
  10. AI summary auto-generates and is stored for the results page
